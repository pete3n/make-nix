make-nix help

Usage:
make <help|install|check|build|switch|all>
[TGT_USER=<user>]
[TGT_HOST=<host>]
[TGT_SYSTEM=<system>]
[CFG_TAGS=<tag1>,<tag2>,...]
[SPECS=<spc1>,<spc2>,...]
[OPTION FLAGS]

Make targets:
  help    - You are here.
  install - Install Nix and/or Nix-Darwin on a bare MacOS or Linux system.
           Will not execute on NixOS or Nix-Darwin managed systems.
  uninstall - Uninstall Nix and/or Nix-Darwin on a bare MacOS or Linux system.
           Will not execute on NixOS or Nix-Darwin managed systems.
  check   - Validate flake configurations for the target user and host.
           Sub-targets: check-home, check-system
  build   - Build Nix closures for home and system configurations.
           Runs check first to prevent configuration mismatches.
           Sub-targets: build-home, build-system
  switch  - Check, build, and activate configurations.
           Sub-targets: switch-home, switch-system
  all     - Full pipeline: install through switch for a bare system.

Configuration parameters (autodetected from current system if not set):
  TGT_USER    - Target username. Defaults to current user.
             Override to build a configuration for a different user.
  TGT_HOST    - Target hostname. Defaults to current hostname.
             Override to build a configuration for a different host.
  TGT_SYSTEM  - Target system tuple. Defaults to current platform.
             Valid values: x86_64-linux, aarch64-linux, x86_64-darwin, aarch64-darwin.
             Override to cross-build for a different architecture.
  CFG_TAGS    - Comma-separated tags to customise the Home Manager configuration
             at build time (no spaces). Example: CFG_TAGS=server,minimal
  SPECS       - Comma-separated list of NixOS specialisation configurations
             to build (no spaces). Example: SPECS=egpu,powersave

  Configuration is stored in make-attrs/system/<user>@<host>.nix or
  make-attrs/home-alone/<user>@<host>.nix and committed to git automatically.
  On subsequent runs, existing attribute files are read and updated with any new values.

Option flags (boolean — assign any truthy value to enable):
Truthy values: 1 y Y yes Yes YES on On ON true True TRUE
Falsey values: 0 n N no No NO off Off OFF false False FALSE

Install flags (install|all):
  USE_DETERMINATE=true  - Install Nix using the Determinate Systems installer.
                         Default installer is the official NixOS installer.

  INSTALL_DARWIN=true   - Install Nix-Darwin for MacOS.

  USE_HOMEBREW=true     - Install Homebrew (MacOS only).

  SINGLE_USER=true      - Install Nix in single-user mode.
                         Only supported with the default installer.

  USE_CACHE=true        - Use additional binary cache substituters defined in
                         make.env as NIX_CACHE_URLS (comma-separated, no spaces).

  USE_KEYS=true         - Trust additional public keys defined in
                         make.env as TRUSTED_PUBLIC_KEYS (comma-separated).

Configuration flags (check|build|switch|all):
  DRY_RUN=true          - Evaluate the configuration without building or activating it.
                         Passes --dry-run to Nix. No outputs will be produced.

  HOME_ALONE=true       - Configure for a system running Home Manager without
                         NixOS or Nix-Darwin. Autodetected if not set.

  USE_HOMEBREW=true     - Enable Homebrew package options in a Nix-Darwin configuration.

  USE_CACHE=true        - Use additional binary cache substituters (see above).

  USE_KEYS=true         - Trust additional public keys (see above).

Universal flags (any target):
  KEEP_LOGS=true        - Preserve logs after operations instead of cleaning up.
                         Useful for debugging. Log paths are printed at startup.

Usage examples:
  - Install Nix using the default installer in single-user mode:
    make install SINGLE_USER=true

  - Install Nix using the Determinate Systems installer and Nix-Darwin on MacOS:
    make install USE_DETERMINATE=true INSTALL_DARWIN=true

  - Validate flake configurations for the current user and host:
    make check

  - Validate only the home-manager configuration for the current user:
    make check-home

  - Build and activate the Home Manager configuration for the current user
    on a standalone system (no NixOS or Nix-Darwin):
    make switch-home HOME_ALONE=true

  - Build and activate the current system configuration; autodetect all settings:
    make switch-system

  - Build and activate the Home Manager configuration for user sam on host xps-15,
    with tags 'debian' and 'server', targeting aarch64-linux:
    make switch-home TGT_USER=sam TGT_HOST=xps-15 \
      TGT_SYSTEM=aarch64-linux CFG_TAGS=debian,server HOME_ALONE=true

  - Dry-run evaluation of both system and home configurations; autodetect all settings:
    make switch DRY_RUN=true

  - Full bootstrap of a bare system: install Nix and activate all configurations:
    make all
